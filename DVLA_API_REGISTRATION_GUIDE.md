## 📋 Overview

The DVLA Vehicle Enquiry Service (VES) allows businesses to retrieve vehicle information using a vehicle registration number. This is essential for auto-populating vehicle details in your garage management system.

---

## 🎯 What You'll Get

With DVLA VES API access, you can retrieve:
- ✅ Vehicle make and model
- ✅ Color
- ✅ Fuel type
- ✅ Engine size (CC)
- ✅ Year of manufacture
- ✅ Date of first registration
- ✅ MOT expiry date
- ✅ Tax status and expiry
- ✅ CO2 emissions
- ✅ Vehicle type approval

## 📝 Eligibility Requirements

### ✅ You Must Be:
1. **A registered UK business** with:
   - Company Registration Number (Companies House)
   - OR: Sole trader with business bank account
   - OR: Limited company
   - OR: Partnership

2. **Able to demonstrate legitimate use** such as:
   - Garage/workshop servicing vehicles
   - Vehicle parts supplier
   - Insurance provider
   - Fleet management
   - Vehicle history check service
   - Car dealership

3. **Data Protection Compliant:**
   - Must be registered with ICO (Information Commissioner's Office)
   - Have privacy policy in place
   - Handle data responsibly under UK GDPR

### ❌ Not Eligible:
- Individuals (non-business use)
- Businesses outside the UK (unless special arrangements)
- Speculative/data harvesting purposes

## 🔒 Security Best Practices

### **Protect Your API Key**

1. **Never expose in frontend code**
   ```javascript
   // ❌ WRONG - Visible in browser
   const apiKey = 'your-api-key';
   
   // ✅ CORRECT - Server-side only
   // API key stays in .env on server
   ```

2. **Use environment variables**
   ```env
   # .env file (never commit to git)
   DVLA_API_KEY=secret_key_here
   ```

3. **Restrict IP access** (if available)
   - Whitelist your server IP in DVLA portal
   - Block unauthorized access

4. **Rotate keys periodically**
   - Request new key every 6-12 months
   - Update immediately if compromised

## 💡 Cost Optimization Tips

### **Reduce API Costs:**

1. **Cache Results**
   - Store vehicle details for 30 days
   - Don't re-query same registration repeatedly
   ```php
   // Check cache first, API second
   $cached = Cache::get("vehicle_{$registration}");
   if (!$cached) {
       $cached = $dvlaService->getVehicleDetails($registration);
       Cache::put("vehicle_{$registration}", $cached, 30*24*60); // 30 days
   }
   ```

2. **Validate Registration Format First**
   - Don't call API with obviously invalid registrations
   - UK format: AA11AAA or A111AAA or AA11 AAA
   ```php
   if (!preg_match('/^[A-Z]{2}[0-9]{2}\s?[A-Z]{3}$/', $reg)) {
       return 'Invalid format - no API call needed';
   }
   ```

3. **User Confirmation**
   - Show user the registration before API call
   - "Is AB12 CDE correct? Lookup will use 1 credit"

4. **Batch Processing**
   - If importing multiple vehicles, do offline
   - Use spreadsheet upload with human verification

## 🆘 Troubleshooting

### **Application Rejected?**

**Common Reasons:**
- ❌ Missing ICO registration
- ❌ Unclear business purpose
- ❌ Incomplete documentation
- ❌ Suspected non-legitimate use

**Solution:**
- Address the issues mentioned
- Provide more documentation
- Clarify your business purpose
- Reapply with corrections

---

### **API Calls Failing?**

**Check:**
1. API key is correct in .env
2. Account has sufficient credit
3. Registration format is valid
4. API endpoint URL is correct
5. Server can access external APIs (firewall)

**Test with cURL:**
```bash
curl -X POST "https://driver-vehicle-licensing.api.gov.uk/vehicle-enquiry/v1/vehicles" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"registrationNumber":"AB12CDE"}'
```

---

### **Unexpected Charges?**

**Monitor Usage:**
- Check portal for API call logs
- Look for automated processes calling API
- Verify caching is working
- Check for retry loops on errors
